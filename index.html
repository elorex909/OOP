Chapter 1: Review On Pointers

This chapter reviews C++ fundamentals, focusing on how memory and pointers work, which is essential for understanding objects.

Key Concepts

  • Memory Allocation: Every variable is stored at a unique memory address
  • Address Operator (&): Use to get the memory address of a variable
  • Pointer Variable: A special variable that stores a memory address
  • Indirection Operator (*): Use to access the value at a pointer's address
  • Pointer Arithmetic: Pointers move by the size of their data type when incremented

Code Examples

int num = 25;
int *p;          // Pointer declaration
p = #        // p points to num's address

cout << *p;      // Output: 25 (value at address)
cout << p;       // Output: 0x4a00 (address)

// Pointer arithmetic
int vals[] = {4, 7, 11};
int *valptr = vals;
cout << *(valptr + 1);  // Output: 7

Common Bugs

  • Uninitialized Pointers: Pointers without valid addresses
  • NULL Pointer References: Accessing pointers set to 0
  • Invalid Pointer References: Using pointers that don't point to valid memory blocks
Key Takeaway: Pointers provide powerful memory management capabilities but require careful handling to avoid bugs. Understanding pointers is crucial for effective OOP programming.

Chapter 2: Review on Functions

This chapter reviews how to create and use functions, another C++ fundamental required for OOP.

Purpose of Functions

  • Organization: Break large problems into smaller, manageable pieces
  • Reusability: Write code once and call it multiple times
  • Abstraction: Hide implementation details
  • Testing: Easier to test isolated functionality
  • Extensibility: Easy to extend program functionality

Parameter Passing

// Pass by value (default)
int addition(int a, int b) {
    return a + b;
}

// Pass by reference
void duplicate(int &a, int &b, int &c) {
    a *= 2;
    b *= 2; 
    c *= 2;
}

// Function overloading
int operate(int a, int b) { return a * b; }
float operate(float a, float b) { return a / b; }

// Default parameters
int divide(int a, int b = 2) {
    return a / b;
}

Advanced Features

  • Function Overloading: Same function name, different parameters
  • Default Parameters: Predefined values for optional parameters
  • Variable Scope: Local variables are only accessible within their function
  • Function Prototypes: Declare functions before defining them
Key Takeaway: Functions enable "divide and conquer" strategy, making code more modular, maintainable, and reusable. They are essential building blocks for OOP methods.

Chapter 3: Review on Structures

This chapter reviews struct, the C-style data structure that is the direct ancestor of the C++ class.

Structure Fundamentals

  • Structure (struct): Groups related variables under a single name
  • Compound Data Type: Members can have different types and lengths
  • Dot Operator (.): Access members of struct variables
  • Arrow Operator (->): Access members through pointers
  • Arrays of Structures: Create arrays of struct types

Structure Implementation

struct movies_t {
    string title;
    int year;
} mine, yours;

// Accessing members
mine.title = "2001 A Space Odyssey";
mine.year = 1968;

// Pointer to structure
movies_t *pMovie = &mine;
cout << pMovie->title;  // Output: 2001 A Space Odyssey

// Array of structures
movies_t films[10];
films[0].year = 1999;

// Nested structures
struct address {
    string street;
    string city;
};

struct employee {
    string name;
    address emp_address;
};

Memory Layout

  • Memory Allocation: Variables created when structure is instantiated
  • Size Calculation: Sum of all member sizes (with padding)
  • Memory Efficiency: Members stored in contiguous memory blocks
  • Pointer Operations: Same arithmetic rules apply to structure pointers
Key Takeaway: Structures are the foundation that evolved into C++ classes, introducing the concept of grouping related data together. They demonstrate the basic principles of encapsulation that are central to OOP.

Chapter 4: Structured Programming vs OOP

This chapter introduces why OOP exists by comparing it to the older "structured" or "procedural" paradigm.

Programming Paradigms

  • Structured Programming: Focuses on functions and procedures
  • Object-Oriented Programming: Focuses on objects that model real-world entities
  • Imperative Programming: Programmer instructs how to change state
  • Declarative Programming: Programmer declares what result is wanted

Structured Programming Limitations

// Structured Programming Example
// Data and functions are separate

struct Player {
    string name;
    int score;
    int position_x;
    int position_y;
};

// Separate functions that operate on Player data
void movePlayer(Player &p, int x, int y) {
    p.position_x = x;
    p.position_y = y;
}

void updateScore(Player &p, int newScore) {
    p.score = newScore;
}

void drawPlayer(Player p) {
    // Drawing logic here
}

// Problems: 
// - Functions and data are separate
// - No encapsulation
// - Hard to reuse
// - Difficult to maintain in large projects

OOP Advantages

  • Encapsulation: Combines data and functions into a single unit
  • Abstraction: Hides complex implementation details
  • Inheritance: Promotes code reusability
  • Polymorphism: Enables flexible code design
  • Real-world Modeling: Better represents real-world entities

OOP Implementation

// Object-Oriented Programming Example
// Data and functions are combined in a class

class Player {
private:
    string name;
    int score;
    int position_x;
    int position_y;
    
public:
    // Constructor
    Player(string n, int x, int y) {
        name = n;
        position_x = x;
        position_y = y;
        score = 0;
    }
    
    // Member functions (methods)
    void move(int x, int y) {
        position_x = x;
        position_y = y;
    }
    
    void updateScore(int newScore) {
        score = newScore;
    }
    
    void draw() {
        // Drawing logic here
    }
    
    // Getter methods
    string getName() { return name; }
    int getScore() { return score; }
};

// Benefits:
// - Data and behavior are encapsulated
// - Easy to reuse
// - Better organization
// - Easier maintenance
Key Takeaway: OOP models the real world better by combining data and behavior into objects, making large projects more manageable, maintainable, and reusable compared to structured programming.

Chapter 5: Basic Concepts of OOP

This chapter defines the four main pillars (foundational concepts) of OOP.

The Four Pillars of OOP

  • Encapsulation: Binding data and functions together in a class
  • Abstraction: Hiding implementation details from the user
  • Inheritance: Creating new classes from existing ones
  • Polymorphism: One interface with multiple implementations

Class and Objects

// Class: Blueprint for creating objects
class HumanBeing {
protected:
    string name;
    int age;
    
public:
    // Constructor
    HumanBeing(string n, int a) {
        name = n;
        age = a;
    }
    
    // Member functions
    void walk() { 
        cout << name << " is walking" << endl;
    }
    
    void talk() {
        cout << name << " is talking" << endl;
    }
    
    void eat() {
        cout << name << " is eating" << endl;
    }
};

// Objects: Instances of the class
HumanBeing ahmed("Ahmed", 25);  // Object 1
HumanBeing mariam("Mariam", 22);  // Object 2

ahmed.walk();  // Output: Ahmed is walking
mariam.talk();  // Output: Mariam is talking

Inheritance Example

// Base class (Parent)
class HumanBeing {
protected:
    string name;
    int age;
    
public:
    HumanBeing(string n, int a) : name(n), age(a) {}
    
    void walk() { cout << name << " is walking" << endl; }
    void talk() { cout << name << " is talking" << endl; }
};

// Derived classes (Children)
class Male : public HumanBeing {
public:
    Male(string n, int a) : HumanBeing(n, a) {}
    
    void shave() { 
        cout << name << " is shaving" << endl;
    }
};

class Female : public HumanBeing {
public:
    Female(string n, int a) : HumanBeing(n, a) {}
    
    void applyMakeup() {
        cout << name << " is applying makeup" << endl;
    }
};

// Using inheritance
Male ahmed("Ahmed", 25);
Female mariam("Mariam", 22);

ahmed.walk();      // Inherited from HumanBeing
ahmed.shave();     // Specific to Male
mariam.talk();      // Inherited from HumanBeing
mariam.applyMakeup(); // Specific to Female

Polymorphism Example

// Base class
class Animal {
public:
    virtual void sound() {  // Virtual function for polymorphism
        cout << "Animal makes a sound" << endl;
    }
};

// Derived classes
class Dog : public Animal {
public:
    void sound() override {
        cout << "Woof! Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() override {
        cout << "Meow! Meow!" << endl;
    }
};

class Cow : public Animal {
public:
    void sound() override {
        cout << "Moo! Moo!" << endl;
    }
};

// Polymorphism in action
void makeSound(Animal &animal) {
    animal.sound();  // Same interface, different implementations
}

int main() {
    Dog dog;
    Cat cat;
    Cow cow;
    
    makeSound(dog);  // Output: Woof! Woof!
    makeSound(cat);  // Output: Meow! Meow!
    makeSound(cow);  // Output: Moo! Moo!
    
    return 0;
}
Key Takeaway: The four pillars of OOP work together to create flexible, maintainable, and reusable code. Encapsulation protects data, abstraction simplifies complexity, inheritance promotes reuse, and polymorphism enables flexibility.

Chapter 6: Classes in C++

This chapter focuses on the practical C++ implementation of classes, especially constructors and destructors.

Constructors and Destructors

  • Constructor: Special function called when object is created
  • Destructor: Special function called when object is destroyed
  • Constructor Overloading: Multiple constructors with different parameters
  • Default Constructor: Provided by compiler if no constructor is defined
  • Copy Constructor: Creates a copy of an existing object

Constructor Implementation

class CRectangle {
private:
    int *width, *height;
    
public:
    // Default constructor
    CRectangle() {
        width = new int(5);
        height = new int(5);
    }
    
    // Parameterized constructor
    CRectangle(int w, int h) {
        width = new int(w);
        height = new int(h);
    }
    
    // Copy constructor
    CRectangle(const CRectangle &other) {
        width = new int(*other.width);
        height = new int(*other.height);
    }
    
    // Destructor
    ~CRectangle() {
        delete width;
        delete height;
        cout << "Destructor called: Memory freed" << endl;
    }
    
    // Member function
    int area() {
        return (*width) * (*height);
    }
    
    // Getters
    int getWidth() { return *width; }
    int getHeight() { return *height; }
};

// Using constructors
CRectangle rect1;           // Default constructor
CRectangle rect2(3, 4);     // Parameterized constructor
CRectangle rect3(rect2);    // Copy constructor

cout << rect1.area();  // Output: 25
cout << rect2.area();  // Output: 12
cout << rect3.area();  // Output: 12

Getters and Setters

class Circle {
private:
    double radius;
    string color;
    
public:
    // Constructor with default values
    Circle(double r = 1.0, string c = "red") {
        radius = r;
        color = c;
    }
    
    // Getters (accessors)
    double getRadius() {
        return radius;
    }
    
    string getColor() {
        return color;
    }
    
    // Setters (mutators) with validation
    void setRadius(double r) {
        if (r > 0) {
            radius = r;
        } else {
            cout << "Error: Radius must be positive" << endl;
        }
    }
    
    void setColor(string c) {
        color = c;
    }
    
    // Member function
    double getArea() {
        return radius * radius * 3.1416;
    }
};

// Using getters and setters
Circle circle(5.0, "blue");
cout << circle.getArea();  // Output: 78.54

circle.setRadius(10.0);
cout << circle.getArea();  // Output: 314.16

circle.setRadius(-5);      // Error: Radius must be positive

Information Hiding and Encapsulation

  • Private Members: Accessible only within the class
  • Public Interface: Controlled access through getters and setters
  • Data Validation: Setters can validate input before modifying data
  • Implementation Hiding: Internal details hidden from users
  • Controlled Access: Prevents direct modification of internal state
Key Takeaway: Constructors ensure proper object initialization, destructors prevent memory leaks, and getters/setters provide controlled access to private data. These features work together to implement proper encapsulation and create robust, maintainable classes.
